## What changed

Claude Code, Anthropic's command-line coding agent, released version 2.1.216 on July 20, 2026.

**New setting:**
- `sandbox.filesystem.disabled` lets you turn off filesystem isolation (the sandbox that restricts which files Claude Code can read or write) while still keeping network access controls in place.

**Fixes:**
- A performance bug where "message normalization" (internal processing of the conversation history) got slower and slower as a session went on, because its cost grew quadratically with the number of turns. This caused multi-second stalls and slow session resumes in long-running sessions.
- Auto mode (a mode that replaces manual permission prompts with automatic safety checks) was wrongly denying commands after an OAuth authentication token expired or rotated mid-session.
- Claude Code on the web would re-ask a question and silently drop your answer if the session had been idle for a few minutes.
- Background agent sessions that were resumed reverted to the default agent instead of keeping their original agent's prompt and tool restrictions.
- Subagents running in an isolated git worktree (a separate working copy of a repository) could redirect git commands into the shared checkout using flags like `git -C`, `--git-dir`, or the `GIT_DIR`/`GIT_WORK_TREE` environment variables, breaking the isolation.
- Workflow saves and scheduled-task writes could follow a symlink placed at `.claude`, which could redirect writes outside the project folder.
- Cloud sessions could drop the message you were sending if the session's container restarted mid-turn.
- Right-to-left text (Arabic, Hebrew, Persian) rendered in the wrong order in VS Code when mixed with English or code.

**Smaller improvements:**
- `/fork`'s confirmation message is now a single line showing the new session's name, its `claude attach` ID, and a note if the copy shares your file checkout.
- The `/ultrareview` "diff too large" error now shows the configured size limit, the measured diff size, and the largest contributing files.
- `/context` now shows an explicit warning when a conversation goes over the context window (the maximum amount of text Claude can consider at once).
- Updated the bundled dataviz skill (used for building charts) with palette and chart-guidance fixes.

## Why it matters

The message-normalization fix matters most for anyone running long Claude Code sessions — the fix removes a slowdown that got worse the longer a session ran, which was making some sessions stall for multiple seconds at a time and slowing down resumes. The two symlink/git-redirect fixes are security-relevant: both could let file writes or git operations escape their intended sandbox or worktree without the user noticing. The new `sandbox.filesystem.disabled` setting is useful if you want network restrictions but don't need (or want) filesystem isolation for your workflow.

## How to use it

- Update Claude Code normally: `claude update`, or through your package manager.
- To disable filesystem sandboxing while keeping network controls, add `"sandbox": {"filesystem": {"disabled": true}}` to your settings.
- No action is needed for the other fixes — they apply automatically after updating.

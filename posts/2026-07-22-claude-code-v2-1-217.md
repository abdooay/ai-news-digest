## What changed

Claude Code, Anthropic's command-line coding agent, released version 2.1.217 on July 21, 2026.

**New limits on subagents:**
- Subagents (helper agents Claude Code spawns to do part of a task) are now capped at 20 running concurrently by default, so a single message can't fan out an unbounded number of background agents. Override this with the `CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS` environment variable.
- Subagents no longer spawn their own nested subagents by default. Set `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH` to allow deeper nesting.
- `--max-budget-usd` (a dollar spending cap for a session) now actually stops background subagents: once the cap is hit, new subagent spawns are denied and any subagents still running in the background are halted. Previously it didn't stop already-running background work.

**Other new features:**
- Emoji shortcode autocomplete in the prompt input — type `:heart:` to insert ❤️, or `:hea` to see suggestions. Turn it off with the `emojiCompletionEnabled` setting.
- Claude Code now warns you if transcript writes are failing (for example, the disk is full) or if session saving is off because of an inherited environment variable, instead of silently losing your transcript.

**Fixes:**
- A memory leak: when an MCP (Model Context Protocol) tool's output was truncated, the full untruncated result stayed in memory for the rest of the session.
- Windows auto-updates that failed could leave `claude.exe` missing entirely; failed updates now restore the previous working executable automatically.
- Background session isolation didn't fully resolve symlinked working directories, which could let a session read or write outside its intended workspace folder.
- Auto-compact (automatic conversation trimming) never triggered for Claude Opus 4.8 on Amazon Bedrock, and running `/compact` manually failed once a session was already over the limit.
- Corporate mTLS, TLS-verification, OAuth scope, and proxy settings were being ignored in Claude Desktop sessions.
- Background shells (started with `/background` or by exiting a running session) could become impossible to stop, especially on Windows under heavy load.
- A `CLAUDE.md` or `SKILL.md` file with a path pattern containing many brace groups (like `{a,b,c,...}`) could make the CLI hang or get killed for using too much memory at startup; brace expansion is now capped.
- `--resume`, `--continue`, and `/resume` failed with an error when a saved transcript had a malformed attachment entry.
- Remote Control (viewing a session from another device) didn't show a pending permission prompt to viewers who connected after the prompt appeared.
- The transcript preview shown while attaching to a starting background session now leaves the same small gap as the live layout does, so the screen doesn't jump when the session takes over.

**Smaller changes:**
- Footer PR badge links are now clickable even when the terminal's hyperlink support can't be detected, such as over SSH or inside tmux. Set `FORCE_HYPERLINK=0` to opt out.
- The login-expiry warning now appears 3 days before expiry instead of 5.
- The tip suggesting the frontend-design plugin now shows at most 3 times total, instead of repeating indefinitely.

## Why it matters

The subagent limits are the most consequential change here: without a default cap, a single prompt could in theory spawn dozens of background agents, driving up cost and load with no warning. The fix to `--max-budget-usd` closes a real gap — previously, hitting your budget cap didn't stop subagents that were already running in the background, so costs could keep accruing after you thought you'd capped them. The symlink and brace-expansion fixes are small but close real ways a session could escape its intended sandbox or crash at startup.

## How to use it

- Update with `claude update`, or through your package manager.
- To allow more than 20 concurrent subagents, set `CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS` to a higher number.
- To let subagents spawn their own subagents again, set `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH`.
- To try emoji autocomplete, type `:` followed by a shortcode name in the prompt input; disable it with `"emojiCompletionEnabled": false` in settings if you don't want it.
- No action is needed for the other fixes — they apply automatically after updating.
